## 使用顺序

1. 先阅读 `review-findings.md`，理解问题证据。
2. 按 `remediation-plan.md` 的 Task R1～R7 顺序修复。
3. 每个修复任务遵循红灯测试、最小实现、全量回归、独立提交。
4. 不应在 R1～R4 未完成时继续扩展界面或增加新 Agent。
